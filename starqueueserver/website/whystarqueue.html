Why StarQueue?

StarQueue programming can start within seconds and requires only sending an HTTPS POST.

By contrast, many other message queues are:
  • too complex
  • heavyweight
  • challenging to install
  • difficult to configure
  • have client software that requires special language specific drivers
  • have client software that tends to be out of date of poorly maintained

Instead of needing custom client drivers, StarQueue uses HTTP long polling which is effective for queue receivers because it works well over standard Internet infrastructure and can be programmed with any modern programming language.
